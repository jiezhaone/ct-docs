# 匯入學員

支援 CSV 及 Excel（.xlsx / .xls）格式批次匯入學員。新版匯入採用**兩階段流程**：先預覽差異、再確認匯入，避免不小心覆寫或重複建立資料。

## 匯入流程總覽

1. **上傳檔案** — 系統解析內容並向後端查詢每筆資料的狀態。
2. **預覽比對** — 每一列會被分類為以下其中之一：

   | 狀態 | 說明 | 是否匯入 |
   |------|------|----------|
   | 🟢 **新增** | 學員編號在系統中不存在 | ✅ 會新增 |
   | 🔵 **更新** | 學員編號已存在於本精舍，且姓名／法名有變動 | ✅ 會更新 |
   | ⚪ **無變動** | 學員編號已存在於本精舍，且資料完全相同 | ❌ 不處理 |
   | 🟠 **他精舍衝突**（CT 為「他精舍衝突」，DM 為「他中心衝突」） | 學員編號存在於其他精舍／中心 | ❌ 不匯入，僅提示 |
   | 🔴 **資料錯誤** | 欄位缺失、精舍名稱不正確、或檔案內有重複編號 | ❌ 無法匯入 |

3. **確認匯入** — 按下「確認匯入」後，系統以**整批交易**方式寫入；若任何一列出錯，會**整批回滾**不寫入任何資料，並列出每筆錯誤原因供排查。

::: tip 預覽表格功能
- 上方「狀態分頁」可依狀態（全部／新增／更新／無變動／他精舍衝突／資料錯誤）快速切換檢視。
- 「更新」的列會以 `舊值 → 新值` 的格式顯示姓名／法名的差異，方便確認。
- 有「資料錯誤」時，「確認匯入」按鈕會被停用，需先修正檔案後重新上傳。
:::

## 法師（DM）匯入

### 檔案格式

欄位順序如下：

| 欄位 | 說明 | 必填 |
|------|------|------|
| 學員編號 | 學員編號 | 是 |
| 姓名 | 學員姓名 | 是 |
| 法名 | 學員法名 | 否 |

::: tip 提示
第一行若為標題列（包含「student_no」、「學員」等字樣），系統會自動略過。
:::

### 操作步驟

1. 進入「學員管理」頁面，點擊「匯入檔案」
2. 點擊「下載範本」取得格式正確的 Excel 範本
3. 點擊「選擇檔案」上傳 CSV 或 Excel
4. 檢視預覽表格，確認每一列的分類狀態是否符合預期
5. 點擊「確認匯入（新增 X、更新 Y）」

### 匯入結果

- **成功**：顯示實際新增與更新的筆數。
- **失敗**：顯示失敗總數，所有資料已回滾（未寫入），並在下方列出每筆錯誤的學員編號與錯誤訊息。

## CT 管理員匯入

::: warning CT 管理員專屬
CT 管理員可跨精舍匯入，每一列必須指定所屬精舍名稱。
:::

### 檔案格式

欄位順序如下：

| 欄位 | 說明 | 必填 |
|------|------|------|
| 精舍名稱 | 所屬精舍的名稱（需與系統一致） | 是 |
| 學員編號 | 學員編號 | 是 |
| 姓名 | 學員姓名 | 是 |
| 法名 | 學員法名 | 否 |

::: warning 精舍名稱需完全一致
若精舍名稱與系統中的名稱不符（含空白、簡繁體差異），該列會被標記為「資料錯誤」而無法匯入。建議先從「匯出全部」下載現有資料作為精舍名稱對照。
:::

### 操作步驟

與 DM 流程相同，但預覽表格會多一欄「精舍名稱」，且狀態中的衝突類別會顯示具體的精舍名稱（例如：「已存在於『普台精舍』，不會匯入」）。

### 判斷邏輯

- **更新**：學員編號存在，且位於檔案指定的同一精舍；
- **他精舍衝突**：學員編號存在，但位於**不同精舍**，此列不會匯入，僅作為提醒避免重複建立；
- **資料錯誤**：精舍名稱不存在於系統、欄位缺失、或檔案內學員編號重複。
